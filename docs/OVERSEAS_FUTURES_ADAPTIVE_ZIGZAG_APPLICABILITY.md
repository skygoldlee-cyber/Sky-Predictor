# Adaptive ZigZag 기반 변동성 모델의 해외선물 매매 적용 가능성 검토

## 1. 개요

Adaptive ZigZag는 ATR 기반 동적 임계값과 ER(Efficiency Ratio) 적응형 필터링을 사용하는 스윙 탐지 지표입니다. 현재 한국 시장(KOSPI/KP200)에 최적화되어 있습니다.

## 2. 해외선물 특성 분석

### 2.1 시장 차이점

| 특성 | 한국 선물 (KP200) | 해외선물 (ES, NQ, GC 등) |
|------|------------------|------------------------|
| 거래 시간 | 09:00-15:35 (장중) | 24시간 거래 |
| 틱 사이즈 | 0.25pt | 상품별 다름 (0.25, 0.5, 1.0 등) |
| 통화 | KRW | USD |
| 변동성 | 장초반/장마감 집중 | 런던/뉴욕 세션 집중 |
| 거래량 | 장중 집중 | 세션별 편차 큼 |
| 데이터 소스 | eBEST API | Interactive Brokers, CQG 등 |

### 2.2 기술적 차이점

- **시간대**: 한국 시간 기준 `early_session_start_time: "09:00"` 설정이 해외선물에는 부적합
- **ATR 파라미터**: 한국 시장 변동성에 맞춰 튜닝된 파라미터는 해외선물에 직접 적용 불가
- **피보나치 레벨**: 가격대/통화 차이로 인한 백분율 기반 레벨 재조정 필요

## 3. 적용 가능성 분석

### 3.1 장점 (적용 가능)

1. **ATR 기반 적응형 필터링**: 변동성에 따라 동적 임계값 조정 → 해외선물 변동성에 대응 가능
2. **ER(Efficiency Ratio) 필터링**: 추세/횡보 구분 → 시장 상태 기반 스윙 탐지 유용
3. **클러스터링 기능**: 근접 피봇 병합 → 잡음 필터링에 효과적
4. **시간대 테이블 지원**: `session_min_wave_atr_ratio_table`로 세션별 파라미터 조정 가능
5. **ATR 기반 필터링**: `use_atr_based_filtering=True`로 파동 크기 필터링 가능

### 3.2 제약사항 (수정 필요)

1. **시간대 설정**: 한국 시간 기준 설정이 해외선물에 부적합
   - `early_session_start_time: "09:00"` → 런던 세션/뉴욕 세션 기준으로 변경 필요
   - `session_min_wave_atr_ratio_table`에 해외선물 세션 시간 추가 필요

2. **ATR 파라미터 재튜닝**: 한국 시장 기본값(`atr_multiplier: 1.5`, `atr_period: 14`)은 해외선물에 부적합
   - ES (S&P 500): 변동성 낮음 → ATR 배수 증가 필요
   - GC (Gold): 변동성 높음 → ATR 배수 감소 필요
   - CL (Crude Oil): 세션별 변동성 차이 큼 → 세션별 파라미터 필요

3. **통화/틱 사이즈**: 현재 백분율 기반 계산은 통화 차이에 영향받지 않으나, 절대 가격 기반 로직 수정 필요

4. **데이터 소스**: eBEST API 대신 해외선물 데이터 소스 연동 필요

## 4. 수정 권장사항

### 4.1 시간대 설정 수정

```python
# 해외선물 (예: ES)용 설정
early_session_start_time: str = "14:30"  # 뉴욕 세션 오픈 (한국 시간)
early_session_end_time: str = "15:00"    # 뉴욕 세션 초반
early_session_atr_multiplier_max: float = 6.0  # 변동성 큰 세션 오픈 시

# 세션별 파라미터 테이블
session_min_wave_atr_ratio_table = [
    ("14:30", "16:00", 0.8),  # 뉴욕 세션 오픈 - 변동성 큼
    ("16:00", "21:00", 0.6),  # 뉴욕 세션 중반
    ("21:00", "06:00", 0.4),  # 아시아 세션 - 변동성 작음
    ("06:00", "14:30", 0.5),  # 유럽 세션
]
```

### 4.2 ATR 파라미터 상품별 튜닝

| 상품 | atr_multiplier | atr_period | min_wave_atr_ratio |
|------|----------------|------------|-------------------|
| ES (S&P 500) | 2.0 | 20 | 0.6 |
| NQ (Nasdaq 100) | 2.5 | 20 | 0.7 |
| GC (Gold) | 1.2 | 14 | 0.4 |
| CL (Crude Oil) | 1.5 | 14 | 0.5 |

### 4.3 데이터 소스 연동

현재 eBEST API 기반 데이터 로드를 해외선물 브로커 API(IB, CQG 등)로 변경 필요:

```python
# 기존 한국 시장 데이터 로드
df = load_korean_futures_data(symbol="FV301")  # KP200

# 해외선물 데이터 로드 (예시)
df = load_overseas_futures_data(symbol="ES", exchange="CME")
```

## 5. 백테스트 및 검증 방법

### 5.1 백테스트 데이터 확보

- Interactive Brokers Historical Data
- CQG Historical Data
- TradingView Data Export

### 5.2 검증 지표

1. **스윙 탐지 정확도**: 실제 고점/저점과 탐지된 피봇 일치율
2. **거짓 신호 비율**: 추세 반전 없이 피봇이 변경되는 비율
3. **지연도**: 실제 고점/저점 탐지까지 걸리는 봉 수
4. **수익성**: 스윙 기반 매매 전략 수익률

## 6. 결론

### 6.1 적용 가능성: **가능 (수정 필요)**

Adaptive ZigZag의 핵심 알고리즘(ATR 기반 필터링, ER 적응형 필터링, 클러스터링)은 해외선물에도 적용 가능합니다. 다만 시간대 설정, ATR 파라미터, 데이터 소스 등 상품별 튜닝이 필요합니다.

### 6.2 우선 순위

1. **데이터 소스 연동**: 해외선물 데이터 로드 기능 구현
2. **시간대 설정 재조정**: 해외선물 세션 기준 시간대 파라미터 수정
3. **ATR 파라미터 튜닝**: 상품별 변동성에 맞춘 파라미터 최적화
4. **백테스트 검증**: 실제 데이터로 성능 검증 및 파라미터 미세조정

### 6.3 추가 고려사항

- **세션별 변동성**: 런던/뉴욕 세션 변동성이 크므로 세션별 파라미터 필수
- **경제 지표 발표 시간**: FOMC, NFP 등 주요 지표 발표 시간 전후 변동성 대응 필요
- **통화 헤지**: USD 기반 상품의 환율 변동성 고려 필요
