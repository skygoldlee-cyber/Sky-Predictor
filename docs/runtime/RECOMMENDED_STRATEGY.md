# RECOMMENDED_STRATEGY.md — 제약조건 하 최선의 매매로직 추천

대상: KOSPI200 연결지수선물(90199999), 주간세션 08:45–15:45, 당일청산, 250일 분봉.
근거: 본 검토 전 과정(비용·승수 반영, 당일청산, purged walk-forward, train/test,
Sharpe 표준오차, 베타 베이스라인, 다중검정/과적합 분석)에서 실증된 결과.

---

## 0. 한 줄 추천

**리스크관리된 롱-바이어스 인트라데이("롱-또는-플랫")** — 매일 시가 롱 진입·종가 청산,
확정 하락추세일엔 플랫(현금), **숏은 하지 않음**, 변동성 기반 사이징 + 손절.
이것은 *알파가 아니라 베타(intraday 롱 드리프트) 수확 + 다운사이드 보호*다.

---

## 1. 왜 이게 최선인가 (확립된 사실)

1. **250일로는 알파를 확립할 수 없다.** 단일 전략 Sharpe SE ≈ ±1.0(250일).
   119개 탐색 시 *참 엣지 0*인데도 운으로 나오는 최고 Sharpe 평균 ≈ 2.6.
   피봇 in-sample best(~2.0)는 이 노이즈 기대치 이하 → 우연으로 설명됨.
2. **OOS에서 살아남은 유일한 신호 = intraday 롱 드리프트(베타).** 비용 반영·train/test 후
   상시 롱 test Sharpe ≈ 1.25(안정). 그 외(피봇·레짐 MA)는 베타를 SE 밖에서 못 이김.
3. **숏 leg는 음(−)의 기대값 + 거대한 변동성** → 제거가 통계적으로 우월.
4. **레짐 MA 오버레이는 알파를 더하지 못함** — 단, `allow_short=False`(롱-또는-플랫)는
   하락 구간 MaxDD를 약 절반으로 줄이는 *다운사이드 보호* 효과가 있었다.

---

## 2. 추천 로직 스펙

| 항목 | 값 |
|------|-----|
| 세션 | 주간 08:45–15:45, 당일청산(오버나잇 금지) |
| 코어 | 매 거래일 시가 진입 → 종가 청산 (intraday 롱 드리프트 수확) |
| 방향 | bull/중립 → 롱, **확정 bear → 플랫(현금)**, 숏 없음 |
| bear 필터 | 일봉 MA단기<MA장기 (전일 종가 기준, shift(1) → look-ahead 없음). 표준값 MA 20/60 |
| 손절(선택) | 장중 손절: 전일 일봉 ATR×k (예: 1.0) 또는 고정 %. 꼬리손실 캡 |
| 사이징 | 변동성 타깃(일일 목표변동성) + 하드 일일손실 한도. 베타이므로 과레버리지 금지 |
| 파라미터 | **표준값 고정**(MA 20/60, 손절 ~1×ATR). 250일에 Optuna 최적화 금지(과적합) |

구현은 이미 보유: `regime_intraday_daily(..., signal=daily_regime_signal(..., allow_short=False))`
+ `intraday_stop_backtest(..., stop_atr_mult=1.0)`.

---

## 3. 의사결정 규칙 (어느 버전을 쓸지)

1. `baseline_window_dispersion(full_i, bt)` 로 **베타의 최악 구간 MaxDD** 확인.
2. **감내 가능하면 → 플레인 상시 롱**(가장 단순, 오버레이 리스크 없음).
3. **너무 깊으면 → 롱-또는-플랫(bear 플랫)** 추가. 그래도 깊으면 **손절** 추가.
4. 추가 요소는 각각 train→test **1회**만 검증, **MaxDD 감소 vs PnL 손실**로 판단.
   (PnL 우위가 기준이 아님 — 상승장 표본에선 PnL은 베타가 더 클 수 있음)

---

## 4. 검증 게이트 (채택 전 필수 — 도구 이미 보유)

- 비용·슬리피지·승수(25만/pt) 반영, 당일청산.
- train/test 분리 + 신호는 전체에서 1회 계산 후 슬라이스(워밍업 보존).
- **파라미터·손절은 train에서 고정, test는 1회 확인** (test-set 선택 금지).
- 추가 복잡성은 베타를 **Sharpe SE 밖**에서 이겨야 정당화.
- 구간별 dispersion으로 레짐 의존성 점검.

---

## 5. 회피 목록 (실증으로 기각됨)

- ❌ **피봇 반전 로직** — 과적합(train 1.9→test 0.05), 베타도 못 이김. 은퇴.
- ❌ **숏 진입** — 음의 기대값, 고변동.
- ❌ **11파라미터 Optuna 탐색(250일)** — 노이즈 적합. 표준 파라미터 사용.
- ❌ **test로 MA/손절 고르기** — OOS 오염(MA10/30 "우위"는 이 착시였음).
- ❌ **베타를 알파로 착각해 과레버리지** — 수익은 설계상 시장 레짐에 종속.

---

## 6. 기대치 (정직하게)

- "수익 보장" 로직은 없다. 이건 *양의 기대값(베타) + 생존 리스크관리*다.
- 수익은 시장이 오를 때 나오고, 하락장에선 (플랫 스위치로) 손실을 줄이는 구조.
- 진짜 검증은 **더 긴 데이터(하락장 포함) + 수개월 전진(paper) 테스트**다.
  250일 백테스트 통과는 필요조건이지 충분조건이 아니다.

---

## 부록 B. 롱-또는-플랫 리스크 관리 개선 백테스트

**원본**: `LONG_OR_FLAT_IMPROVEMENTS_BACKTEST.md`

### 목적
기존 롱-또는-플랫에 다음 3가지를 추가하여 개선 여부 검증.
- 전일 일봉 ATR 기반 손절/익절
- ATR 기반 포지션 사이징
- 일일 손실 한도

### 벤치마크
| 항목 | 수치 |
|------|------|
| 거래수 | 125 |
| 승률 | 56.00% |
| PnL | +60,310,113 원 |
| Sharpe | 1.493 |
| MaxDD | -31,050,193 원 |

### 개선 버전 최고 조합
- **Sharpe 기준**: SL1.5ATR TP3.0ATR R1% L10M
  - 거래수: 73, 승률: 57.53%, PnL: +38,304,418 원, Sharpe: 2.593, MaxDD: -18,970,963 원
- **PnL 기준**: SL1.0ATR TP2.0ATR R1% L10M
  - 거래수: 88, 승률: 59.09%, PnL: +42,522,011 원, Sharpe: 2.369, MaxDD: -16,698,391 원

### 결론
손절/익절 + ATR 사이징 + 일일 손실 한도를 추가하면 **절대 수익은 기존보다 줄지만, Sharpe는 1.49 → 2.59로 향상되고 MaxDD는 1/2 이하로 감소**한다. 리스크 조정 관점에서는 의미 있는 개선이다.

## 부록 D. 피봇반전 로직 검증

**원본**: `PIVOT_REVERSAL_VERIFICATION.md`

### 한 줄 결론
피봇반전 파이프라인은 **검출부터 체결까지 look-ahead가 없다**. 그러나 홀드아웃 테스트에서 **최적 파라미터조차 베타(장중 상시 롱)를 못 이기고 손실을 낸다**. "엣지 없음"은 코드 버그가 아니라 진짜 신호다.

### Look-ahead 없음 근거
- `HybridAdaptivePivot.update`는 인과적: 극점에서 되돌림 확인 후 후보 등록
- 확정 신호는 최소 1봉 지연 후 발생
- `_calc_er`는 현재 봉을 제외한 과거 종가만 사용
- 진입은 신호 확정 봉의 **다음 봉 시가**에서 실행

### 홀드아웃 비교 (n_trials=60, val_block=21일, test_frac=0.2)
| 전략 | 거래 | 승률 | PnL (원) | Sharpe | MaxDD |
|------|------|------|----------|--------|--------|
| 피봇반전 (best params) | 56 | 50.00% | -8,018,211 | -0.515 | -44,330,271 |
| 베타(상시 롱) | 48 | 52.08% | +27,053,913 | 1.254 | -23,349,005 |
| 롱-또는-플랫 | 34 | 50.00% | +20,989,323 | 1.170 | -23,349,005 |

### 권고
1. **운영 로직 교체 보류**: 피봇반전을 롱-또는-플랫/베타의 대체로 채택하지 않는다.
2. **선택 기준은 nested 프로토콜**: 단일 in-sample Sharpe 폐기. opt↔홀드아웃 분리 + 베타 대비 SE 비교를 디폴트로.
3. **데이터 확장 후 재검증**: 2년+ 데이터에서 홀드아웃이 길어져 SE가 좁아진 뒤에야 "유의하게 베타를 이기는가"가 판별 가능.
