## 1. 개요

본 가이드는 TradeExecutionGate 운영 중 발생할 수 있는 장애 상황을 식별하고, 신속하게 대응하여 서비스 중단을 최소화하기 위한 절차를 제공합니다.

### 목표
- 장애 감지 시간 최소화 (5분 이내)
- 복구 시간 최소화 (30분 이내)
- 장애 재발 방지

### 책임자
- 1차 담당자: 시스템 운영자
- 2차 담당자: 개발팀

## 2. 장애 분류

### 2.1 심각도 등급

| 등급 | 정의 | 응답 시간 | 복구 시간 | 예시 |
|------|------|----------|----------|------|
| **P0** | 서비스 완전 중단, 손실 발생 | 5분 | 30분 | 진입/청산 불가, 잘못된 주문 |
| **P1** | 기능 부분 장애 | 15분 | 2시간 | 특정 조건에서 진입 차단, 로그 누락 |
| **P2** | 성능 저하 | 1시간 | 1일 | 지연 발생, 리소스 사용량 증가 |
| **P3** | 사소한 문제 | 1일 | 1주 | 로그 포맷 오류, 문서 오류 |

### 2.2 장애 유형

#### 2.2.1 진입/청산 관련
- 진입 불가 (신호 무시)
- 잘못된 진입 (조건 미충족 진입)
- 청산 불가 (손실 누적)
- 잘못된 청산 (조건 미충족 청산)

#### 2.2.2 데이터/상태 관련
- 상태 손실 (재시작 시 초기화)
- 이력 저장 실패 (JSONL 미기록)
- 설정 로드 실패 (config.json 오류)

#### 2.2.3 통신/알림 관련
- 텔레그램 알림 전송 실패
- 예측 데이터 수신 실패
- API 연결 실패

#### 2.2.4 리스크 관리 관련
- 리스크 한도 초과 (잘못된 계산)
- 슬롯 할당 오류
- Trailing Stop 오작동

## 3. 일반적인 장애 조치 절차

### 3.1 장애 감지

1. **로그 모니터링**
   ```bash
   # 실시간 로그 확인
   tail -f logs/trading.log | grep -E "ERROR|WARN|CRITICAL"
   ```

2. **텔레그램 알림 확인**
   - 진입/청산 알림 수신 여부
   - 에러 메시지 확인

3. **시스템 상태 확인**
   ```bash
   # 프로세스 상태
   ps aux | grep python
   # 메모리/CPU 사용량
   top
   ```

### 3.2 장애 보고

1. **장애 등록**
   - 등급 분류 (P0/P1/P2/P3)
   - 발생 시각 기록
   - 증상 상세 기록

2. **관련자 통지**
   - P0: 즉시 전화/메시지
   - P1: 15분 이내 메시지
   - P2/P3: 이메일

### 3.3 임시 조치

1. **서비스 중단** (P0의 경우)
   ```python
   # 텔레그램 명령으로 트레이딩 중지
   /trade_gate off
   ```

2. **로그 수집**
   ```bash
   # 최근 로그 백업
   cp logs/trading.log logs/trading.log.backup.$(date +%Y%m%d_%H%M%S)
   ```

3. **상태 저장**
   ```bash
   # 현재 상태 백업
   cp trade_history/*.jsonl backup/
   ```

### 3.4 근본 원인 분석

1. **로그 분석**
   - 에러 메시지 추적
   - 스택 트레이스 확인
   - 전후 맥락 파악

2. **코드 검토**
   - 관련 로직 확인
   - 최근 변경 사항 확인

3. **데이터 검증**
   - config.json 유효성 확인
   - 이력 데이터 무결성 확인

### 3.5 복구

1. **수정 적용**
   - 핫픽스 배포 (필요 시)
   - 설정 수정

2. **테스트**
   ```bash
   # 단위 테스트 실행
   python -m pytest tests/test_trade_gate.py -v
   ```

3. **서비스 재개**
   ```python
   # 텔레그램 명령으로 트레이딩 재개
   /trade_gate on
   ```

### 3.6 사후 분석

1. **장애 보고서 작성**
   - 원인
   - 영향
   - 조치 내용
   - 재발 방지책

2. **개선 사항 반영**
   - 코드 개선
   - 모니터링 강화
   - 문서 업데이트

## 4. 특정 장애 시나리오 및 조치 방법

### 4.1 진입 불가 (P0)

#### 증상
- 신호 수신 중에도 진입하지 않음
- 로그에 "진입 차단" 메시지

#### 원인
- 리스크 한도 초과 (연속 손실, 일일 손실)
- 슬롯 소진
- 설정 오류 (min_confidence, min_prob_buy 등)
- 시간대 제한 (장 마감)

#### 조치 방법
1. **상태 확인**
   ```python
   # 텔레그램 명령으로 상태 확인
   /trade_status
   ```

2. **리스크 한도 확인**
   - `consecutive_losses` 확인
   - 일일 손실 확인

3. **설정 확인**
   ```json
   // config.json
   {
     "trade_gate": {
       "enabled": true,
       "max_consecutive_losses": 3,
       "max_daily_loss_pt": 5.0
     }
   }
   ```

4. **조치**
   - 리스크 한도 초과: 설정 조정 또는 대기
   - 슬롯 소진: 다음 슬롯 대기 또는 설정 조정
   - 설정 오류: config.json 수정 후 재시작

### 4.2 청산 불가 (P0)

#### 증상
- 포지션 보유 중 청산되지 않음
- 손실이 계속 누적됨

#### 원인
- check_close 호출 누락
- 청산 조건 오류
- 예외 발생으로 청산 로직 중단

#### 조치 방법
1. **즉시 조치**
   ```python
   # 수동 청산 (API 직접 호출)
   # 또는 트레이딩 중지
   /trade_gate off
   ```

2. **로그 확인**
   ```bash
   grep "check_close\|execute_close" logs/trading.log
   ```

3. **포지션 확인**
   ```python
   # 현재 포지션 상태 확인
   # trade_history viewer 사용
   python trading/history_viewer.py
   ```

4. **복구**
   - 수동 청산 (브로커 API)
   - 로그 수정 후 재시작
   - 예외 처리 강화

### 4.3 잘못된 진입 (P1)

#### 증상
- 조건 미충족에도 진입함
- 잘못된 사이드/가격으로 진입

#### 원인
- 조건 로직 오류
- 신호 데이터 오염
- 설정 오류

#### 조치 방법
1. **즉시 조치**
   ```python
   # 진입 중지
   /trade_gate off
   ```

2. **포지션 정리**
   - 잘못된 포지션 수동 청산

3. **원인 분석**
   - 진입 로그 확인
   - 신호 데이터 검증
   - 설정 확인

4. **복구**
   - 로직 수정
   - 단위 테스트 추가
   - 설정 수정

### 4.4 이력 저장 실패 (P2)

#### 증상
- JSONL 파일에 기록되지 않음
- 파일 생성되지 않음

#### 원인
- 디스크 공간 부족
- 권한 문제
- 경로 오류
- 인코딩 오류

#### 조치 방법
1. **디스크 확인**
   ```bash
   df -h
   ```

2. **권한 확인**
   ```bash
   ls -la trade_history/
   ```

3. **설정 확인**
   ```json
   {
     "trade_gate": {
       "history_save_enabled": true,
       "history_dir": "trade_history"
     }
   }
   ```

4. **복구**
   - 디스크 공간 확보
   - 권한 수정
   - 경로 수정

### 4.5 텔레그램 알림 전송 실패 (P2)

#### 증상
- 진입/청산 알림 수신 안 됨
- 에러 로그에 전송 실패 기록

#### 원인
- 토큰 만료
- 네트워크 문제
- 채트 ID 오류
- API 제한

#### 조치 방법
1. **토큰 확인**
   - config.json의 텔레그램 토큰 확인

2. **네트워크 확인**
   ```bash
   ping api.telegram.org
   ```

3. **채트 ID 확인**
   - 올바른 채트 ID인지 확인

4. **복구**
   - 토큰 갱신
   - 네트워크 문제 해결
   - 채트 ID 수정

### 4.6 리스크 한도 오작동 (P1)

#### 증상
- 정상 상황에서도 리스크 한도 초과로 진입 차단
- 잘못된 손실 계산

#### 원인
- 계산 로직 오류
- 상태 초기화 실패
- 설정 오류

#### 조치 방법
1. **상태 확인**
   ```python
   /trade_status
   ```

2. **계산 검증**
   - 일일 손실 계산 확인
   - 연속 손실 카운트 확인

3. **설정 확인**
   ```json
   {
     "trade_gate": {
       "max_consecutive_losses": 3,
       "max_daily_loss_pt": 5.0
     }
   }
   ```

4. **복구**
   - 로직 수정
   - 상태 초기화
   - 설정 수정

### 4.7 Trailing Stop 오작동 (P1)

#### 증상
- 의도하지 않은 시점에 청산
- Trailing Stop이 작동하지 않음

#### 원인
- 파라미터 오류
- 조건 로직 오류
- 시간 문제

#### 조치 방법
1. **설정 확인**
   ```json
   {
     "trade_gate": {
       "trailing_stop_enabled": true,
       "trailing_stop_activation_pt": 1.0,
       "trailing_stop_distance_pt": 0.5
     }
   }
   ```

2. **로그 확인**
   ```bash
   grep "Trailing Stop" logs/trading.log
   ```

3. **복구**
   - 설정 수정
   - 기능 비활성화
   - 로직 수정
