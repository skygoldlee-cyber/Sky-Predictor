# 레짐 기반 당일 매매 전략 가이드

## 개요

레짐 기반 당일 매매 전략은 시장의 추세(레짐)를 감지하여 상승장에서는 롱, 하락장에서는 숏, 횡보장에서는 진입을 스킵하는 전략입니다. 매일 시가에 진입하여 종가에 청산하는 당일 매매 방식으로, 매일 한 번 이상의 매매 신호를 보장하는 것을 목표로 합니다.

**⚠️ 중요:** 이 문서의 기존 성과 수치는 look-ahead bias가 포함된 결과입니다. 방법론적 결함 수정 후 인과적 구현으로 재검증한 결과는 아래 "방법론적 결함 수정" 섹션을 참조하세요.

## 전략 로직

### 레짐 감지

**MA 크로스오버 방식**
```python
if MA_20 > MA_60:
    regime = "bull"  # 상승장 → 롱 진입
elif MA_20 < MA_60:
    regime = "bear"  # 하락장 → 숏 진입
else:
    regime = "neutral"  # 횡보장 → 진입 스킵
```

**ADX 기반 방식**
```python
if ADX >= 25.0:
    if MA_20 > MA_60:
        regime = "bull"  # 상승장 + 강한 추세 → 롱 진입
    elif MA_20 < MA_60:
        regime = "bear"  # 하락장 + 강한 추세 → 숏 진입
else:
    regime = "neutral"  # 약한 추세 → 진입 스킵
```

### 당일 매매

```python
# 상승장: 롱 진입
if regime == "bull":
    시가 진입 → 종가 청산
    수익 = (종가 - 시가) - 비용

# 하락장: 숏 진입
elif regime == "bear":
    시가 진입 → 종가 청산
    수익 = (시가 - 종가) - 비용

# 횡보장: 진입 스킵
else:
    진입 스킵
```

## 주요 파라미터

### 레짐 감지 파라미터

- `regime_method`: 레짐 감지 방식 ("ma" 또는 "adx")
- `ma_short`: 단기 이동평균 기간 (기본값: 20)
- `ma_long`: 장기 이동평균 기간 (기본값: 60)
- `adx_threshold`: ADX 임계값 (기본값: 25.0)

### 필터 파라미터

- `filter_atr_ratio`: ATR 기반 변동성 필터 (기본값: 0.0, 필터 없음)
- `gap_threshold`: 갭 크기 필터 (기본값: 0.0, 필터 없음)
- `intraday_reversal`: 당일 중간 레짐 역전 감지 (기본값: False)

import pivot_optuna_v2 as pv

# 백테스트 설정

    multiplier=250000,  # KOSPI200 futures multiplier
    commission_pct_per_side=0.00015,  # 0.015% commission
    slippage_ticks_per_side=1,  # 1 tick slippage
    tick_size=0.05,  # KOSPI200 futures tick size
    entry_on="next",  # 다음 바 진입
    annualization=252,  # 연간 거래일

    session_boundary_hour=8,  # 08:00 기준으로 거래일 구분
    direction_mode="both",  # 롱/숏 모두
)

# 레짐 기반 당일 매매
result = pv.regime_based_intraday(
    df,
    bt,
    regime_method="ma",
    ma_short=20,
    ma_long=60,
    adx_threshold=25.0,
    filter_atr_ratio=0.0,
    gap_threshold=0.0,
    intraday_reversal=False,

## 방법론적 결함 수정 및 최종 검증 결과

### 문제점 식별

기존 문서의 성과 수치는 다음과 같은 방법론적 결함이 있었습니다:

1. **Look-ahead bias**: 분봉 데이터에서 MA를 계산하여 일봉 레짐 감지에 사용
2. **Test-set selection bias**: TEST Sharpe가 가장 높은 MA 조합을 선택하여 OOS 검증 무효화
3. **Warmup artifact**: MA20/60이 TEST에서 0건으로 나타난 것은 워밍업 절단 문제
4. **Sharpe 해석 불일치**: test>>train을 "우수"로 해석하다가 test<<train을 "과적합"으로 해석

### 인과적 구현 (regime_intraday_v2.py)

다음과 같은 방법론적 수정을 적용한 인과적 구현을 제공:

1. **일봉 기반 레짐 감지**: 일봉 OHLC에서 MA 계산 후 shift(1)로 look-ahead 제거
2. **전체 기간 신호 계산**: 워밍업 절단 방지를 위해 전체 기간 일봉에서 1회 계산 후 슬라이스
3. **TRAIN 기준 선택**: TEST Sharpe가 아닌 TRAIN Sharpe로 MA 조합 선택
4. **Sharpe 표준오차 동반**: 짧은 TEST 기간에서 차이가 노이즈인지 판정

### 방법론적 교훈

1. **Look-ahead bias 제거**: 분봉 MA 대신 일봉 MA 사용 + shift(1)
2. **Test-set 선택 금지**: TRAIN 기준으로 파라미터 선택
3. **Warmup 절단 방지**: 전체 기간에서 1회 계산 후 슬라이스
4. **Sharpe 표준오차 동반**: 짧은 TEST 기간에서 통계적 유의성 판정
5. **구간별 안정성 확인**: 단일 홀드아웃이 운인지(레짐 의존) 판단

이 방법론적 수정이 "MA10/30이 베타를 이긴다"던 신호가 노이즈로 드러나게 하여, 잘못된 채택을 막았습니다.
