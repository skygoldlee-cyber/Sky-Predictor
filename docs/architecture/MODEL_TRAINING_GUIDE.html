모델 학습 가이드¶

이 문서는 Transformer 모델 학습을 위한 데이터 수집, 데이터셋 구축, 모델 학습 절차를 상세히 설명합니다.

  2. 학습한 가중치 사용 시 기대 효과
  3. 데이터 수집
  4. 데이터셋 구축
  5. 모델 학습
  6. 학습 후 관리
  7. 재학습 절차
  8. 주의사항

1. 학습 개요¶

학습 목적¶

Transformer 모델 학습을 통해 Rule-based 신호와 결합하여 승률을 향상시킵니다.

모드 승률 신호 빈도 설명
Rule-based만 65% 15~20회/일 Adaptive Indicator만 사용
Transformer + Rule-based 70~75% 9~12회/일 모델 예측 + Rule-based 결합

학습 흐름¶

┌─────────────────────────────────────────────────────────┐
│              모델 학습 흐름                               │
│                                                          │
│  실시간 틱 데이터 수집 (1~3개월)                          │
│           ↓                                              │
│  리플레이 파일 (ticks_replay_*.jsonl.gz)                │
│           ↓                                              │
│  데이터셋 구축 (data_builder.py)                         │
│           ↓                                              │
│  학습 데이터셋 (dataset.npz)                             │
│           ↓                                              │
│  모델 학습 (train.py)                                    │
│           ↓                                              │
│  가중치 파일 (prediction/weights/patch_tst_multiscale.pt)│
│           ↓                                              │
│  실시간 예측 파이프라인에 적용                            │
└─────────────────────────────────────────────────────────┘

2. 학습한 가중치 사용 시 기대 효과¶

승률 향상¶

모드 승률 향상 폭 설명
Rule-based만 65% 기준 Adaptive Indicator만 사용
Transformer + Rule-based 70~75% +5~10% 모델 예측으로 필터링

승률 향상 이유: - Transformer 예측으로 거짓 신호 필터링 - Rule-based 신호의 신뢰도 점검 - 노이즈 감소

신호 빈도 변화¶

모드 신호 빈도 감소 폭 설명
Rule-based만 15~20회/일 기준 모든 신호 실행
Transformer + Rule-based 9~12회/일 -30~40% 고품질 신호만 실행

신호 감소 이유: - Transformer 예측 확신도 낮은 신호 제외 - Rule-based 신호와 모델 예측 불일치 시 제외 - 더 정교한 신호 선택

신호 품질 향상¶

Rule-based만: - 노이즈 많은 신호 포함 - 거짓 신호 빈번 - 횡보 구간에서 오신호

Transformer + Rule-based: - 노이즈 필터링 - 거짓 신호 감소 - 추세 구간에서 정확한 신호

구체적 기대 효과¶

현재 65% 승률 기준:

항목 Rule-based만 Transformer + Rule-based 향상
승률 65% 70~75% +5~10%
일일 신호 15~20회 9~12회 -30~40%
승리 횟수 10~13회 6~9회 -4~4회
패배 횟수 5~7회 3~4회 -2~3회
순수익 기대 양성 더 높은 양성 증가

멀티스케일 효과 추가¶

멀티스케일 5분봉 추가 시:

항목 기본 +5분봉 향상
승률 65% 68~70% +3~5%
신호 감소 기준 -30~40% 감소

멀티스케일 5분 + 15분봉 추가 시:

항목 기본 +5분+15분봉 향상
승률 65% 70~72% +5~7%
신호 감소 기준 -50~60% 큰 감소

최종 기대 효과 (조합)¶

최적 조합: - Transformer 학습 완료 - 멀티스케일 5분봉 활성화 - Rule-based 결합

항목 기대값
승률 70~75%
일일 신호 9~12회
신호 품질 높음
노이즈 낮음

요약¶

학습한 가중치 사용 시: - 승률: 65% → 70~75% (+5~10%) - 신호 빈도: 15~20회/일 → 9~12회/일 (-30~40%) - 신호 품질: 거짓 신호 감소, 정교한 신호 - 순수익: 증가 기대

추천: 현재 65% 승률에서 Transformer 학습 + 멀티스케일 5분봉 조합으로 승률 70~75% 달성 가능.

3. 데이터 수집¶

데이터 수집 기간¶

기간 만기주 경험 권장 여부 이유
2주일 옵션 만기주 0회 ⚠️ 최소 만기주 패턴 미포함
1개월 옵션 만기주 1회 ✅ 최소 만기주 경험 포함
3개월 옵션 3회 + 선물 1회 ✅ 권장 다양한 만기주 패턴
6개월 옵션 6회 + 선물 2회 ✅ 최적 완전한 패턴 학습

권장: 최소 1개월, 권장 3개월

데이터 수집 절차¶

1단계: 매일 데이터 수집

python main.py --include-options --out-ticks ticks_replay_YYYYMMDD.jsonl.gz

2단계: 데이터 자동 저장 - 틱 데이터 자동 저장 - 분봉 OHLCV 자동 집계 - 데이터 위치: 현재 디렉토리

3단계: 데이터 크기 관리 - 하루 데이터: ~20MB (압축) - 1개월 데이터: ~600MB - 3개월 데이터: ~1.8GB

리플레이 파일 형식¶

파일명: ticks_replay_YYYYMMDD_HHMMSS.jsonl.gz

내용: - FC0: 선물 체결 - FH0: 선물 호가 - OC0: 옵션 체결 - OH0: 옵션 호가 - JIF: 지수

4. 데이터셋 구축¶

데이터셋 구축 명령어¶

기본 명령어:

python -m prediction.data_builder \
  --files ticks_replay_*.jsonl.gz \
  --out dataset_1month.npz \
  --seq-len 60 --horizon 5 \
  --multiscale-5m

파라미터 설명:

파라미터 설명 기본값 권장값
--files 리플레이 파일 목록 필수 ticks_replay_*.jsonl.gz
--out 출력 데이터셋 파일명 dataset_5m.npz dataset_1month.npz
--seq-len 시퀀스 길이 (분) 60 60
--horizon 예측 horizion (분) 5 5
--multiscale-5m 5분봉 멀티스케일 피처 추가 false true
--config config.json 경로 config.json config.json

멀티스케일 피처¶

기본 (1분봉만): - Feature dimension: 87 - OB: 11, CD: 9, OPT: 13, Adaptive: 43, Time: 11

1분 + 5분봉: - Feature dimension: 95 (+8 증가) - Multiscale: 8 (MS5)

1분 + 5분 + 15분봉: - Feature dimension: 103 (+16 증가) - Multiscale: 16 (MS5 + MS15)

데이터셋 출력¶

출력 파일: dataset_1month.npz

내용: - X: 입력 피처 (shape: [N, seq_len, feature_dim]) - y: 타겟 (shape: [N]) - metadata: 메타데이터 (JSON)

5. 모델 학습¶

모델 학습 명령어¶

기본 명령어:

python training/train.py --config config.json

학습 설정¶

config.json 설정:

{
  "training": {
    "batch_size": 32,
    "epochs": 100,
    "learning_rate": 0.001,
    "weight_decay": 0.0001
  }
}

학습 과정¶

  1. 데이터셋 로드
  2. dataset_1month.npz 로드

  3. Train/Validation 분리

  4. 모델 초기화

  5. PatchTST 모델 초기화

  6. Feature dimension 확인

  7. 학습 루프

  8. Epoch 반복
  9. Loss 계산

  10. 가중치 업데이트

  11. 검증

  12. Validation set 평가
  13. Best model 저장

학습 결과¶

출력 파일: - prediction/weights/patch_tst_multiscale.pt - 학습된 가중치

학습 시간: - CPU: 1~2일 - GPU: 2~4시간

6. 학습 후 관리¶

가중치 파일 배치¶

위치: prediction/weights/patch_tst_multiscale.pt

자동 배치: 학습 완료 후 자동 저장

실시간 예측 적용¶

적용 방법: 1. 가중치 파일 존재 확인 2. main.py 실행 3. Transformer 모델 자동 로드 4. Transformer + Rule-based 결합 모드 작동

동작 확인:

[FEATURE_DIM] feature_dim=95 multiscale_5m=True
[MODEL] Loading weights: prediction/weights/patch_tst_multiscale.pt

7. 재학습 절차¶

재학습 필요 시점¶

필수 재학습: - 새로운 피처 추가 - Feature dimension 변경 - 모델 구조 변경

주기적 재학습: - 월간: 성능 모니터링 후 승률 저하 시 - 분기별: 시장 구조 변화 확인 - 반기별: 장기 데이터 축적 후

이벤트 기반 재학습: - 큰 시장 사건 (금융 위기, 정책 변화) - 레짐 변화 (횡보 → 추세 전환) - 성능 급격 저하

재학습 절차¶

1단계: 새 데이터 추가

python main.py --include-options --out-ticks ticks_replay_YYYYMMDD.jsonl.gz

2단계: 데이터셋 재구축

python -m prediction.data_builder \
  --files ticks_replay_*.jsonl.gz \
  --out dataset_updated.npz \
  --seq-len 60 --horizon 5 \
  --multiscale-5m

3단계: 모델 재학습

python training/train.py --config config.json

4단계: 가중치 교체 - 기존 가중치 백업 - 새 가중치 배치 - 성능 검증

승률 모니터링¶

모니터링 방법: - 일일 승률 기록 - 주간 승률 평균 - 월간 승률 평균

재학습 기준: - 승률 65% 미만: 재학습 고려 - 승률 60% 미만: 필수 재학습
